# ERC-4907: Rental NFT Standard

## Overview

ERC-4907 extends ERC-721 with a dual ownership model, supporting both **permanent ownership** and **temporary usage rights**. It enables NFT rental and leasing without transferring ownership.

**Key Innovation:** Separate owner from user rights

---

## Core Concepts

### Dual Ownership Model

```
Owner: Person/entity who permanently owns the NFT
  - Can transfer or sell the NFT
  - Receives royalties on secondary sales
  - Can revoke user rights

User: Person/entity with temporary rights to use the NFT
  - Can use/interact with the NFT
  - Cannot transfer or sell
  - Rights expire after rental period
```

### Time-Based Access

```
NFT: Sword #123
  Owner: Alice
  User: Bob
  Expiration: Block 1000000 (expires when block number exceeds this)

At Block 1000000:
  - Bob loses user rights
  - Alice can rent again or transfer
```


## Implementation Patterns

### Pattern 1: Basic Rental

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "@openzeppelin/contracts/token/ERC721/ERC721.sol";

contract RentalNFT is ERC721 {
    mapping(uint256 => address) private users;
    mapping(uint256 => uint64) private userExpires;
    
    event UpdateUser(
        uint256 indexed tokenId,
        address indexed user,
        uint64 expires
    );
    
    function userOf(uint256 tokenId) public view returns (address) {
        if (uint64(block.number) >= userExpires[tokenId]) {
            return address(0);
        }
        return users[tokenId];
    }
    
    function userExpires(uint256 tokenId) public view returns (uint256) {
        return userExpires[tokenId];
    }
    
    function setUser(
        uint256 tokenId,
        address user,
        uint64 expires
    ) public {
        require(msg.sender == ownerOf(tokenId), "Not owner");
        require(expires > block.number, "Expiration in past");
        
        users[tokenId] = user;
        userExpires[tokenId] = expires;
        
        emit UpdateUser(tokenId, user, expires);
    }
}
```

### Pattern 2: Rental with Payment

```solidity
contract RentalMarketplace {
    struct RentalListing {
        address owner;
        uint256 pricePerDay;
        bool available;
    }
    
    mapping(uint256 => RentalListing) public listings;
    
    function rentNFT(
        address nftContract,
        uint256 tokenId,
        uint256 days
    ) external payable {
        RentalListing memory listing = listings[tokenId];
        
        // Verify payment
        uint256 totalPrice = listing.pricePerDay * days;
        require(msg.value >= totalPrice, "Insufficient payment");
        
        // Calculate expiration
        uint64 expires = uint64(block.number + (days * 12960)); // ~1 day in blocks
        
        // Set user
        IERC4907(nftContract).setUser(tokenId, msg.sender, expires);
        
        // Transfer payment to owner
        payable(listing.owner).transfer(totalPrice);
    }
}
```

### Pattern 3: Lending Protocol

```solidity
contract LendingProtocol {
    struct Loan {
        address borrower;
        uint256 amount; // USDC or similar
        uint64 expiresAt;
        bool repaid;
    }
    
    mapping(uint256 => Loan) public loans;
    
    function borrowWithCollateral(
        address nftContract,
        uint256 tokenId,
        uint256 loanAmount,
        uint64 duration
    ) external {
        // Verify collateral value
        require(getNFTValue(nftContract, tokenId) >= loanAmount * 2, "Insufficient collateral");
        
        // Set user rights to protocol
        IERC4907(nftContract).setUser(tokenId, address(this), uint64(block.number) + duration);
        
        // Track loan
        loans[tokenId] = Loan({
            borrower: msg.sender,
            amount: loanAmount,
            expiresAt: uint64(block.number) + duration,
            repaid: false
        });
        
        // Send loan amount
        USDC.transfer(msg.sender, loanAmount);
    }
    
    function repayLoan(uint256 tokenId) external {
        Loan storage loan = loans[tokenId];
        require(msg.sender == loan.borrower, "Not borrower");
        require(!loan.repaid, "Already repaid");
        
        // Receive repayment
        USDC.transferFrom(msg.sender, address(this), loan.amount);
        
        // Clear user (borrower can use NFT again)
        IERC4907(nftContract).setUser(tokenId, address(0), uint64(block.number));
        
        loan.repaid = true;
    }
}
```

## Integration with ERC-721

### ERC-4907 vs Pure ERC-721

```solidity
// ERC-721 only: Transfer for temporary use
function rentERC721(address to, uint256 tokenId) {
    transferFrom(msg.sender, to, tokenId);
    // Problems: Owner changes, can't force return
}

// ERC-4907: Rent with automatic expiration
function rentERC4907(address to, uint256 tokenId, uint64 expires) {
    setUser(tokenId, to, expires);
    // Benefits: Owner unchanged, automatic expiration
}
```

## Time-Based Expiration

### Block-Based vs Timestamp-Based

```solidity
// Option 1: Block number (more reliable)
uint64 expires = uint64(block.number + 250000); // ~1 month on Ethereum

// Option 2: Timestamp (easier for users)
uint64 expires = uint64(block.timestamp + 30 days);

// Considerations:
// - Block-based: Consistent across all chains
// - Timestamp-based: Human-readable, but can be manipulated by miners
```

### Duration Calculation

```solidity
// Calculate blocks for duration
function calculateExpiration(uint256 days) public view returns (uint64) {
    // Ethereum: ~12 seconds per block, ~7200 blocks per day
    uint256 totalBlocks = days * 7200;
    return uint64(block.number + totalBlocks);
}

// Calculate duration from expiration
function calculateDuration(uint64 expires) public view returns (uint256 days) {
    require(expires > block.number, "Already expired");
    uint256 blockDifference = expires - block.number;
    return blockDifference / 7200;
}
```

## Real-World Scenarios

### Scenario 1: Gaming Item Rental

```
Alice owns: Legendary Sword
Bob wants to use it for a tournament
Terms:
  - Rental: 3 days
  - Price: 0.5 ETH

Process:
1. Alice lists Sword For Rent at 0.5 ETH / 3 days
2. Bob pays 0.5 ETH
3. Alice calls setUser(tokenId, Bob, expirationBlock)
4. Bob uses Sword in games for 3 days
5. After 3 days, Bob loses user rights
6. Alice can re-rent or transfer Sword
```

### Scenario 2: Virtual Real Estate

```
Alice owns: Digital Land Parcel
Bob wants to develop on it temporarily

Terms:
  - Lease: 1 year
  - Rent: 10 USDC/month

1. Bob pays 120 USDC upfront
2. Alice sets Bob as user for 365 days
3. Bob develops structures/content on land
4. After 365 days, lease expires
5. Alice can keep/sell parcel, or re-lease
```

### Scenario 3: Collateralized Lending

```
Alice owns: Bored Ape NFT (worth $50,000)
Alice needs cash: $10,000 loan

1. Alice deposits NFT as collateral
2. Protocol sets itself as user (prevents transfers)
3. Alice receives $10,000 USDC
4. Alice has 30 days to repay
5. Option A: Repay $10,000 + interest → Get NFT back
6. Option B: Default → Protocol liquidates NFT
```

## Security Considerations

1. **Expiration Precision** - Account for block time variability
2. **User Revocation** - Only owner can change user rights
3. **Transfer Safety** - Clarify what happens on owner transfer
4. **Approval Integration** - Define approval flow for setUser()
5. **Reentrancy** - Protect against reentrancy in rental logic

## Best Practices

### 1. Input Validation

```solidity
function setUser(
    uint256 tokenId,
    address user,
    uint64 expires
) public {
    require(msg.sender == ownerOf(tokenId), "Not owner");
    require(user != address(0), "Invalid user");
    require(expires > block.number, "Invalid expiration");
    
    // Set user
}
```

### 2. Event Emission

```solidity
// Always emit events for important state changes
emit UpdateUser(tokenId, user, expires);
```

### 3. Emergency Functions

```solidity
// Allow owner to recover NFT in case of bugs
function emergencyStopRental(uint256 tokenId) external onlyOwner {
    setUser(tokenId, address(0), uint64(block.number));
}
```
