# ERC-6551: Token Bound Accounts (TBA) - The Future of NFTs 🔥

## Overview

ERC-6551 enables NFTs to own assets independently. Each NFT gets its own smart contract wallet that can hold tokens, interact with dapps, and participate in transactions—making NFTs **autonomous entities** rather than just static assets.

**Key Innovation:** NFTs become smart contract wallets

---

## Core Concept

### Traditional NFT Model
```
NFT: Sword #123
- Owned by: Alice
- Can be: Transferred, sold, displayed
- Actions: None (passive)
```

### Token Bound Accounts (TBA)
```
NFT: Sword #123
├── Owner: Alice
├── TBA Account: 0x7a...9c (Smart Wallet)
│   ├── Holds: 100 USDC
│   ├── Holds: 5 potions (ERC-1155)
│   ├── Holds: Shield #456 (ERC-721)
│   └── Can Execute: Trades, swaps, contracts
└── Features: Quest rewards, inventory, enchantments
```

---

## Key Components

### 1. Account Proxy
Each NFT has a deterministic account address:
- Same blockchain
- Calculated from: NFT contract + token ID
- Unique and predictable

### 2. Account Implementation
Smart contract wallet logic for TBA

### 3. Account Registry
Maps NFT → Account address

## Key Functions

### Registry Interface

```solidity
interface IERC6551Registry {
    /**
     * Create or retrieve the account for an NFT
     */
    function account(
        address implementation,
        bytes32 salt,
        uint256 chainId,
        address tokenContract,
        uint256 tokenId
    ) external view returns (address);
	
    /**
     * Create account if it doesn't exist
     */
    function createAccount(
        address implementation,
        bytes32 salt,
        uint256 chainId,
        address tokenContract,
        uint256 tokenId,
        uint256 initialValue,
        bytes calldata initData
    ) external returns (address);
}
```

### Account Interface

```solidity
interface IERC6551Account {
    /**
     * Execute a transaction from the account
     */
    function execute(
        address target,
        uint256 value,
        bytes calldata data,
        uint8 operation
    ) external returns (bytes memory);
    
    /**
     * Get the current NFT owner
     */
    function owner() external view returns (address);
    
    /**
     * Get the NFT this account is bound to
     */
    function token() external view returns (address, uint256);
}
```

### Pattern 1: Basic TBA Account

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

interface IERC6551Account {
    receive() external payable {}
    
    function execute(
        address target,
        uint256 value,
        bytes calldata data,
        uint8 operation
    ) external returns (bytes memory) {
        require(msg.sender == owner(), "Unauthorized");
        
        (bool success, bytes memory result) = target.call{value: value}(data);
        require(success, "Execution failed");
        
        return result;
    }
    
    function owner() public view returns (address) {
        address nftContract = getTokenContract();
        uint256 tokenId = getTokenId();
        return IERC721(nftContract).ownerOf(tokenId);
```

### Pattern 2: Game Character with Inventory

```solidity
contract GameCharacter {
    struct Character {
        address nftContract;
        uint256 tokenId;
        string name;
        uint256 level;
        uint256 experience;
    }
    
    mapping(address => Character) public characters;
    
    function createCharacter(
        address nftContract,
        uint256 tokenId,
        string memory name
    ) external {
        address owner = IERC721(nftContract).ownerOf(tokenId);
        require(msg.sender == owner, "Not owner");
        
        characters[getTBA(nftContract, tokenId)] = Character({
            nftContract: nftContract,
            tokenId: tokenId,
            name: name,
            level: 1,
            experience: 0
        });
    }
    
    function addExperience(uint256 amount) external {
        Character storage char = characters[msg.sender];
        require(char.tokenId != 0, "No character");
        
        char.experience += amount;
        
        // Level up logic
        if (char.experience >= 1000) {
            char.level++;
            char.experience = 0;
        }
    }
}
```

## How TBA Works: Step by Step

### Step 1: NFT Minting
```
User mints NFT #123 from contract 0xABC...
NFT belongs to: Alice
```

### Step 2: TBA Generation
```
Registry creates TBA account for:
  - Contract: 0xABC...
  - Token ID: 123
  - Result: 0x7a9c... (deterministic address)
```

### Step 3: Move Assets to TBA
```
Alice sends assets to TBA address:
  - 100 USDC
  - Shield #456 (ERC-721)
  - Potion (ERC-1155)
```

### Step 4: Execute Transactions
```
Alice (NFT owner) calls:
  TBA.execute(target, value, data)
  
TBA executes on behalf of itself:
  - Can swap USDC to ETH
  - Can equip Shield to game contract
  - Can trade items
  - Can participate in contracts
```

### Step 5: Transfer NFT
```
Alice transfers NFT #123 to Bob

Immediately:
  - Bob is new owner
  - Bob can use TBA
  - All TBA assets are still there
  - TBA autonomy follows the NFT
```

```solidity
    implementation,

    tokenId
);
```

```solidity
    // Create it
    registry.createAccount(
        implementation,
        salt,
        chainId,
        nftContract,
        tokenId,
        0,  // initialValue
```

## Security Considerations

1. **Owner Validation** - Always verify NFT owner before permitting actions
2. **Authorization** - Only NFT owner should control TBA
3. **Nonce Management** - Prevent replay attacks
4. **Fallback Handler** - Careful with cross-chain operations
5. **Lost Keys** - TBA can't be recovered if keys lost (NFT still valid)
