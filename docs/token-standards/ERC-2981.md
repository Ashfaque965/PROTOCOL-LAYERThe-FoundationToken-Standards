# ERC-2981: NFT Royalty Standard

## Overview

ERC-2981 is a standard interface for specifying royalty information for NFTs. It enables automatic royalty payments to creators on secondary sales, ensuring creators continue to benefit from their work's appreciation.

**Key Innovation:** Creator rights are standardized and automated

---

## Core Concept

### Royalty Information
- Each token can have associated royalty information
- Royalty recipient and percentage per token ID
- Marketplaces query and respect this standard
- Not enforced on-chain (requires marketplace cooperation)

### Key Principle
Creators receive a percentage of each secondary sale automatically.

---

## Key Functions

### Query Function

```solidity
/**
 * @notice Called with the sale price to determine how much royalty
 * is owed and to whom.
 * @param tokenId - the NFT asset queried for royalty information
 * @param salePrice - the sale price of the NFT asset specified by tokenId
 * @return receiver - address of who should be sent the royalty payment
 * @return royaltyAmount - the royalty payment amount for the given sale price
 */
function royaltyInfo(uint256 tokenId, uint256 salePrice)
    external
    view
    returns (address receiver, uint256 royaltyAmount);
```

## Core Attributes

### Royalty Parameters
1. **Receiver** - Address that receives royalty payments
2. **Royalty Percentage** - Usually 0-10% of sale price
3. **Per-Token Configuration** - Can vary by token ID

## Implementation Patterns

### Pattern 1: Default Royalty

```solidity
uint96 public defaultRoyalty = 500; // 5% (in basis points = 500/10000)
address public royaltyReceiver;

function royaltyInfo(uint256 tokenId, uint256 salePrice)
    external
    view
    returns (address, uint256)
{
    uint256 royaltyAmount = (salePrice * defaultRoyalty) / 10000;
    return (royaltyReceiver, royaltyAmount);
}
```

### Pattern 2: Per-Token Royalty

```solidity
mapping(uint256 => RoyaltyInfo) public tokenRoyalties;

struct RoyaltyInfo {
    address receiver;
    uint96 royaltyPercentage;
}

function royaltyInfo(uint256 tokenId, uint256 salePrice)
    external
    view
    returns (address receiver, uint256 royaltyAmount)
{
    RoyaltyInfo memory info = tokenRoyalties[tokenId];
    if (info.receiver == address(0)) {
        // Use default
        receiver = defaultReceiver;
        royaltyAmount = (salePrice * defaultRoyalty) / 10000;
    } else {
        receiver = info.receiver;
        royaltyAmount = (salePrice * info.royaltyPercentage) / 10000;
    }
}
```

### Pattern 3: Dynamic Royalty

```solidity
// Royalty changes over time
mapping(uint256 => RoyaltyInfo[]) public tokenRoyaltyHistory;

function royaltyInfo(uint256 tokenId, uint256 salePrice)
    external
    view
    returns (address receiver, uint256 royaltyAmount)
{
    RoyaltyInfo[] storage history = tokenRoyaltyHistory[tokenId];
    RoyaltyInfo memory current = history[history.length - 1];
    
    uint256 percentage = current.royaltyPercentage;
    uint256 royalty = (salePrice * percentage) / 10000;
    
    return (current.receiver, royalty);
}
```

## Basis Points System

### Percentage Conversions

```
10000 basis points = 100%
1000 basis points = 10%
500 basis points = 5%
100 basis points = 1%
10 basis points = 0.1%
1 basis point = 0.01%
```

### Calculation

```solidity
// Royalty in basis points (0-10000)
uint96 royaltyPercentage = 500; // 5%

// Calculate royalty amount
uint256 royaltyAmount = (salePrice * royaltyPercentage) / 10000;
```

## Integration with ERC-721/1155

### Combined Implementation

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "@openzeppelin/contracts/token/ERC721/ERC721.sol";
import "@openzeppelin/contracts/token/common/ERC2981.sol";

contract RoyaltyNFT is ERC721, ERC2981 {
    address public owner;
    
    constructor() ERC721("RoyaltyNFT", "RNFT") {}
    
    function mint(address to, uint256 tokenId) public onlyOwner {
        _mint(to, tokenId);
    }
    
    function setTokenRoyalty(
        uint256 tokenId,
        address receiver,
        uint96 royaltyPercentage
    ) public onlyOwner {
        _setTokenRoyalty(tokenId, receiver, royaltyPercentage);
    }
    
    function setDefaultRoyalty(
        address receiver,
        uint96 royaltyPercentage
    ) public onlyOwner {
        _setDefaultRoyalty(receiver, royaltyPercentage);
    }
    
    function supportsInterface(bytes4 interfaceId)
        public
        view
        override(ERC721, ERC2981)
        returns (bool)
    {
        return super.supportsInterface(interfaceId);
    }
}
```

## Marketplace Usage

### How Marketplaces Use Royalty Info

```javascript
// When listing an NFT for sale
const [receiver, royaltyAmount] = await contract.royaltyInfo(tokenId, salePrice);

// Marketplace logic:
// - Send amount to buyer after taking commission
// - Send royaltyAmount to receiver
// - Ensure sum ≤ salePrice

// Example for $1000 sale with 5% royalty and 2.5% fee:
// - Seller receives: $925 (100% - 5% - 2.5%)
// - Royalty receiver: $50 (5%)
// - Marketplace: $25 (2.5%)
```

## Best Practices

### 1. Reasonable Percentages
```solidity
// Good: 3-10%
// Bad: >50%
// Prevents market issues and unsustainable economics
require(royaltyPercentage <= 10000, "Royalty too high");
```

### 2. Careful Receiver Handling
```solidity
// Validate receiver address
require(receiver != address(0), "Invalid receiver");

// Consider using contract with fallback for safety
```

### 3. Platform-Specific Handling
```solidity
// Some platforms have limits
// Example: OpenSea limits to 10%
function setRoyalty(address receiver, uint96 percentage) public {
    // Ensure compatibility
    require(percentage <= 1000, "OpenSea limit: 10%");
    _setDefaultRoyalty(receiver, percentage);
}
```

### 4. Transparent Documentation
```solidity
// Document royalty intent
/**
 * @notice NFTs include 5% royalty to support ongoing development
 * Royalties go to the DAO treasury for community use
 */
```

## Security Considerations

1. **Basis Points Overflow** - Use uint96 to prevent overflow
2. **Receiver Validation** - Verify receiver address validity
3. **Percentage Limits** - Set reasonable maximum limits
4. **Immutable Receiver** - Consider making receiver immutable
5. **Event Logging** - Always emit events for changes
